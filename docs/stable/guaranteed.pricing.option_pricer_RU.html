guaranteed.pricing.option_pricer_RU module¶

class guaranteed.pricing.option_pricer_RU.OptionPricer(grid, N, option, x0, price_support, price_dynamics='add', constraint_set=None, debug_mode=False, ignore_warnings=False, enable_timer=False, profiler_data=None, calc_market_strategies=False, pricer_options={})¶

Bases: object

Represents the numeric solver to the European option pricing problem under the guaranteed approach.

Parameters

  • grid (class:Grid) – Point grid.

  • N (int >= 0) – Investment horizon.

  • option (class:IOption) – Option being priced.

  • x0 (numpy.ndarray) – Vector of current prices.

  • price_support (class:ISetHandler) – Set object representing the support K. Must be compact.

  • price_dynamics ('add' or 'mult') – Additive or multiplicative price dynamics. Default is ‘add’.

  • constraint_set (class:ISetHandler) – Set object representing the trading constraints D.

  • debug_mode (boolean) – If True, debug information is displayed during execution. Default is False.

  • ignore_warnings (boolean) – If True, warnings from the linprog optimization procedures are not displayed. Default is False.

  • enable_timer (boolean) – If True, profiler information will be displayed during execution. Default is False.

  • profiler_data (class:'ProfilerData') – Profiler data, to which the execution timing can be appended to. If None, a new profiler data object will be created. Default is None.

  • calc_market_strategies (boolean) – If True, adverse market strategies at every step will calculated. Not used for pricing. True leads to the slower execution speed. Default is False.

  • pricer_options (dict) – Options for numerical methods.

See also

Grid, IOption, ProfilerData

get_support_set(curr_set, increment)¶

Based on the set of price scenarios at K_{t-1} and the increment dK_t, returns the price scenarios at t. For additive dynamics, returns the set

K_t = {x_{t-1} + y_t, x_{t-1} in K_{t-1}, y_t in dK_t};

For multiplicative dunamics, returns the set

K_t = {x_{t-1} * y_t, x_{t-1} in K_{t-1}, y_t in dK_t};

generate_evaluation_point_lists(p0, dK, N, profiler_data=None)¶

Precalculates the price scenarios for every t to avoid unnecessary evaluations of V_t.

find_u(x, v, z, calc_market_strategies)¶

Returns u(z), see the algorithm.

find_rho(x, v, K_x, convdK_x, calc_market_strategies)¶

Returns the value function value V_t, given V_{t+1} at x_{t+1}.

evaluate()¶

Calculates the option value by backward-reconstructing the value function on a grid. Returns all the intermediate values of the value function Vf at points Vp. Vf[0][0] is the option value.
